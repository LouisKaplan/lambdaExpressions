Lambda Expressions - Short and Sweet

One of the nice benefits of Lambda Expressions is that they're significantly shorter than similar functions:


When theoretically assigning a function to a variable, you might expect it to look like this:

variableForCode = public void doSomething(){
System.out.println("Hello Class!");
}


Except that Lambda Expressions don't need to be delcared as public or private.

variableForCode = void doSomething(){
System.out.println("Hello Class!");
}


We're also going to be referring to this by the variable name, so there's no need need to list a second function name:

variableForCode = void (){
System.out.println("Hello Class!");
}


Also, the compiler in Java8 is smart enough to determine what the return type is, so we can strip that away as well:

variableForCode = (){
System.out.println("Hello Class!");
}


But now your compiler is going to freak out because a lot of things are suddenly missing. We can fix that by indicating that we're using a Lambda Expressions, as denoted by "->":

variableForCode = () -> {System.out.println("Hello Class!");}


And one more optional shortcut: If our function is only one line, we don't even need the braces:

variableForCode = () -> System.out.println("Hello Class!");

Which looks a lot less cumbersome than what we started with:

variableForCode = public void doSomething(){System.out.println("Hello Class!");}




Another example would be a function that mutliplies a parameter by two:

doubleNumberFunction = public int doubler(int a){
return a *2;
}


As a Lamda Expression, it looks more like this:

doubleNumberFunction = (int a) -> a *2;



An expression that adds two numbers might look like this:

addNumbersFunction = public int adder(int a, int b){
return a + b;
}


But as a Lamda Expression:

addNumbersFunction = (int a, int b) -> a + b ;




By now you've surely noticed that all of our above Lambda Expressions are sitting as variables without types, but not for long. In practice, they do have a type - interfaces!
